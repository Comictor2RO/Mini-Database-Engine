============================================================
1) Mini Database Engine (gen SQLite Lite)
============================================================

Un proiect excelent pentru CV, pentru că arată:
    • structuri de date avansate
    • gestionarea memoriei
    • parsing
    • optimizări
    • graphics programming (RayLib GUI)

PROGRES
============================================================

[✓] IMPLEMENTAT:
    • AST complet
        - Statement (baza abstracta)
        - SelectStatement
        - InsertStatement
        - DeleteStatement
        - CreateStatement
        - Condition
        - Columns
    • Lexer
        - TokenType, Token (fișiere separate)
        - Tokenizare completa: keywords, identifiers,
          numbers, strings, operators, punctuation, wildcard
    • Parser

    • Table         (leaga Catalog + PageManager)

    • RayLib GUI

ROADMAP
============================================================

  Saptamana 1:  Table + Engine       -> DB functional end-to-end
  Saptamana 2:  B+ Tree Index        -> cautare rapida
  Saptamana 3:  Page Cache LRU       -> optimizare I/O
  Saptamana 4:  WAL                  -> durabilitate
  Final:        RayLib GUI           -> interfata grafica

============================================================
ARHITECTURA
============================================================

1. SQL Parser
   Primeste un string gen SELECT * FROM users WHERE id = 5
   si il transforma intr-un AST pe care engine-ul il executa.

2. Page-Based Storage
   Fisierul mydb.db este impartit in pagini de 4KB.
   Randurile sunt ambalate in pagini — imita comportamentul
   real al HDD/SSD (citesti/scrii blocuri, nu octeti).

3. B+ Tree Index
   Structura arborescenta pentru cautare rapida dupa cheie
   primara. Fara index: scan complet. Cu B+ Tree: O(log n).

4. Page Cache (Buffer Pool)
   Tine in RAM paginile recent accesate (politica LRU).
   Cache hit -> RAM. Cache miss -> disc + adauga in cache.

5. Write-Ahead Log (WAL)
   Orice modificare e scrisa intai in log, apoi pe disc.
   La crash -> recovery din log. Fundamentul durabilitatii.

RAYLIB GUI
============================================================

  Layout:
  ┌──────────────────────────────────────────────────┐
  │                Mini DB Engine                    │
  ├──────────────────────────────────────────────────┤
  │  ┌────────────────────────────────┐  ┌───────┐  │
  │  │ SELECT * FROM users            │  │  RUN  │  │
  │  └────────────────────────────────┘  └───────┘  │
  ├──────────────────────────────────────────────────┤
  │   id  │  name  │  age                            │
  │  ─────┼────────┼──────                           │
  │   1   │  Ion   │  25    <- citit din mydb.db     │
  │   2   │  Ana   │  30                             │
  ├──────────────────────────────────────────────────┤
  │  [LOG] ✅ Query executed in 0.002ms – 2 rows     │
  │  [LOG] ❌ ERROR: Table 'x' does not exist        │
  └──────────────────────────────────────────────────┘

  Componente:
    • SQL Input field   - text editabil, Enter sau buton RUN
    • Buton RUN         - executa query-ul curent
    • Results table     - grid dinamic bazat pe schema tabelei
                          date citite LIVE din mydb.db
    • Log panel         - timestamp + mesaje
                          verde = succes | rosu = eroare

  Flow:
    SQL Input -> Parser -> Engine -> PageManager
                                         |
                                    mydb.db (disc)
                                         |
                               rows deserializate -> UI
